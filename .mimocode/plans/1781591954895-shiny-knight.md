# 前端展示 422 错误详情方案（修订版）

## 用户反馈的问题

### 问题 1：前端提示信息重复
```
"新合同信息解析失败: 合同解析失败：输入数据缺少以下字段：「支付方式」"
```
Python 的 `_format_contract_error` 将 ValueError 包装成 `"合同解析失败：{exc}"`，Java 又在其前加上 `"新合同信息解析失败: "`，导致重复。

### 问题 2：日志不显示原始错误
```
contract_inner 失败: 合同解析失败：输入数据缺少以下字段：「支付方式」
```
日志记录的是 `detail`（格式化后），而非原始 exception 消息。

### 问题 3：支付方式补全后仍提示缺少（根因）
用户在文本中写的是 `支付方式：xxx`，但 `ContractParser` 的 keywords 列表中只有 `"付款方式"`，没有 `"支付方式"`。导致解析器提取不到该字段，`determine_payment_method("")` 返回 `""`，随后被 `REQUIRED_FIELDS` 校验拦截。

## 待修改文件及方案

### 1. `python-app/routers/ContractAutoGen.py`

**1a. `_format_contract_error` — 去掉 ValueError 的包装前缀**

- 当前：`KeyError` → `"缺少必要字段：{exc}"`，`ValueError` → 走兜底 `"合同解析失败：{exc}"`
- 修改后：增加 ValueError 的显式处理，直接返回 `str(exc)`（ValueError 消息本身已够用户友好）

```python
def _format_contract_error(exc: Exception) -> str:
    if isinstance(exc, KeyError):
        return f"缺少必要字段：{exc}"
    if isinstance(exc, ValueError):
        return str(exc)
    if isinstance(exc, TypeError):
        return f"字段类型错误：{exc}"
    if isinstance(exc, ValidationError):
        errors = exc.errors()
        fields = [e.get("loc", ["未知"])[-1] for e in errors]
        return f"请求参数校验失败，字段：{', '.join(str(f) for f in fields)}"
    return f"合同解析失败：{exc}"
```

**1b. Logger — 记录原始异常而非格式化后信息**

- 当前：`logger.error(f"contract_inner 失败: {detail}")`
- 修改后：`logger.error(f"contract_inner 失败: {exc}")`

### 2. `python-app/utils/contract_inner_service.py` — 添加 "支付方式" 关键词

在 `ContractParser.__init__` 的 `self.keywords` 列表（第 60 行附近）中，在 `"付款方式"` 之前或之后添加 `"支付方式"`。

```python
self.keywords = [
    ...
    "付款方式",
    "支付方式",
    ...
]
```

### 3. 前端 `frontend/src/pages/admin/NewContract.vue`（已修改，验证无误）

第 507 行已改为：
```javascript
const msg = error?.response?.data?.message || error?.response?.data?.detail || error.message || '插入失败'
```

## 预期效果

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| 用户输入缺少支付方式 | "新合同信息解析失败: 合同解析失败：缺少..." | "新合同信息解析失败: 输入数据缺少以下字段：「支付方式」" |
| 补全了支付方式（包含该字段） | 仍报错（parser 没提取到） | 正常解析 |
| 日志内容 | "contract_inner 失败: 合同解析失败：..." | "contract_inner 失败: 输入数据缺少以下字段：「支付方式」" |

## 验证方式

1. 重启 Python 后端服务
2. 输入包含 `支付方式：电汇` 的合同文本，确认不再提示缺少
3. 输入不包含任何支付方式字段的合同文本，确认前端提示为 `"新合同信息解析失败: 输入数据缺少以下字段：「支付方式」"` 而非重复
4. 查看日志，确认 `ERROR` 级别日志显示原始 exception 消息而非包装后的消息
